
# Important

source of truth of game mechanics and technical architecture decisions is in the ADRs in the ARCHITECTURE folder

changes to code should start with updating/creating relevant ADRs, then updating the code to match the ADRs

or vice versa if code changes exist that need documentation in ADRS, docs need to be updated/aligned/created to match the code changes

the ADRs should be written in a way that they can be read and understood independently of the code, so they should include enough context and explanation to make sense on their own
